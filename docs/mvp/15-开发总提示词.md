# UI 规范补充

开发前必须阅读并遵守 `16-UI设计规范与前端交互范式.md`。前端必须采用企业级 B 端 SaaS 管理后台范式，优先使用 Element Plus 组件，保证资产列表、动态表单、审批待办、生命周期时间线和报表看板具备完整的加载、空数据、错误、无权限和响应式状态。

# 15-开发总提示词

## 1. 用途

本文档用于交给 AI 开发平台或软件开发团队，指导其严格基于 `docs/mvp` 目录实现 IT 资产管理平台。

开发时必须控制范围，按 MVP-0 到 MVP-5 分阶段实现，不得跳阶段，不得提前实现后续复杂能力。

## 2. 项目目标

请实现一个 Web 平台：

**多租户、元数据驱动、生命周期优先的 IT 资产管理平台。**

平台需要支持：

- 多租户。
- 用户、组织、角色、权限。
- 资产类型和动态字段配置。
- 有形资产和无形资产统一管理。
- 机房资产完整生命周期。
- 软件许可证完整生命周期。
- 证书、域名、云资源轻量管理。
- 生命周期状态机和动作 API。
- 轻量审批。
- 导入导出。
- 报表看板。
- 审计、通知、附件、API Token、Webhook。

## 4. 已确认技术栈

```text
后端：Spring Boot 3 + Java 21
数据库：PostgreSQL
缓存：Redis
前端：Vue 3 + TypeScript + Vite
UI：Element Plus
数据库迁移：Flyway
API 文档：OpenAPI / Swagger
认证：JWT + Refresh Token
部署：Docker Compose 起步
```

不得自行切换技术栈。若受平台限制必须调整，必须保持数据模型、API 语义、权限规则和验收标准一致。

## 5. 开发顺序

必须按以下顺序实现：

```text
MVP-0 平台底座
MVP-1 元数据与资产核心
MVP-2 生命周期状态机与动作（不含审批）
MVP-3 审批、权限增强、通知
MVP-4 导入导出、报表、轻量资产扩展
MVP-5 生产化加固与集成
```

每个阶段完成后必须运行测试和构建，并根据 `12-测试用例与验收标准.md` 验收。

## 6. 核心约束

必须遵守：

- 所有租户业务表带 `tenant_id`。
- 平台用户无租户上下文，租户用户不能访问平台管理接口。
- API Token 必须绑定租户和 scopes，数据库只保存哈希。
- 前端传入的 `tenant_id` 不可信。
- 资产状态只能通过生命周期动作接口修改。
- MVP-2 不实现人工审批；MVP-3 开始审批联动生命周期动作。
- 动态字段由元数据驱动，不能写死。
- 热点/唯一字段必须有物理列或数据库索引策略。
- 字段权限必须在后端响应和导出中生效。
- 数据权限必须在后端查询中生效。
- 审批任务只能由任务审批人或符合节点规则的用户处理。
- 软件许可证分配数量不能超过可用数量，并发超额必须返回 409。
- 附件下载必须鉴权。
- 敏感字段必须加密存储、脱敏或隐藏。
- 关键操作必须写审计日志。
- 所有接口必须遵守统一响应信封、分页、错误码和鉴权头约定。

## 7. 最小演示主线

必须保证以下主线可运行：

```text
平台管理员创建租户
-> 租户管理员创建组织、用户、角色
-> 租户管理员配置资产类型、字段、表单、列表
-> 资产管理员创建服务器资产
-> 服务器完成采购、入库、上架、运行、维修、退役、处置
-> 软件管理员创建软件许可证
-> 软件许可证完成采购、授权登记、分配、回收、续费、停用
-> 审批人处理采购、退役、软件授权分配审批
-> 普通用户按权限查看资产
-> 导入导出和报表按权限可用
-> 审计日志和通知可查询
```

## 8. 不得实现的内容

第一版不得实现：

- 旧系统历史数据迁移。
- 完整低代码平台。
- BPMN 流程设计器。
- 云厂商自动同步。
- 自动资产扫描。
- 复杂 CMDB 拓扑图。
- 每租户独立数据库。
- SaaS 计费。

这些只作为后续扩展预留。

## 9. 交付验收

最终交付必须包含：

- 后端代码。
- 前端代码。
- Flyway 数据库迁移。
- 初始化种子数据。
- OpenAPI 文档。
- Docker Compose 本地启动。
- 测试用例。
- 部署文档。
- 备份恢复脚本。
- 每个 MVP 阶段的验收清单。

验收标准以 `12-测试用例与验收标准.md` 为准。

## 10. 第一阶段开发启动提示词

请基于 `docs/mvp` 全部文档，只启动 MVP-0 平台底座开发。先实现工程骨架、Docker Compose、PostgreSQL、Redis、Flyway、平台用户、租户、租户用户、组织、角色、权限、JWT 登录/刷新/退出/改密、平台/租户上下文隔离、健康检查、OpenAPI 和基础审计。不要实现资产、生命周期、审批、导入导出、报表或 Webhook。完成后运行后端测试、前端构建、数据库迁移，并按 `12-测试用例与验收标准.md` 的 MVP-0 用例逐项验收。
