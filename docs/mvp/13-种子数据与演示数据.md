# 13-种子数据与演示数据

## 1. 种子数据目标

种子数据用于：

- 支持本地开发。
- 支持演示。
- 支持自动化测试。
- 支持外部开发平台快速理解业务。

种子数据必须可重复初始化，并支持测试环境隔离。自动化测试应使用独立 tenant code 或事务回滚，不能依赖人工演示数据的执行顺序。

## 2. 默认租户

租户：

```text
tenant_code: demo
tenant_name: 演示租户
status: active
```

测试租户：

```text
tenant_code: demo_b
tenant_name: 演示租户B
status: active
```

`demo_b` 用于跨租户隔离测试。

## 3. 默认组织

组织：

```text
总部
  信息技术部
    运维组
    软件资产组
  财务部
  业务一部
  业务二部
```

## 4. 默认用户

用户：

```text
tenant_admin / 租户管理员
asset_admin / 资产管理员
ops_user / 机房运维人员
license_admin / 软件管理员
procurement_user / 采购人员
maintenance_user / 维修人员
dept_manager / 部门负责人
asset_user / 普通使用人
auditor / 审计员
approver / 审批人
```

所有演示用户初始化时：

```text
must_change_password: true
initial_password: ChangeMe_12345
```

首次登录后只允许访问 `POST /api/v1/auth/change-password`，改密成功后 `must_change_password=false`。

## 5. 默认角色和权限映射

角色：

- `tenant_admin`
- `asset_admin`
- `asset_operator`
- `license_admin`
- `procurement_user`
- `maintenance_user`
- `department_manager`
- `asset_user`
- `auditor`
- `approver`

用户绑定：

| 用户 | 角色 | 组织 | 数据范围 |
|---|---|---|---|
| tenant_admin | tenant_admin | 信息技术部 | all_tenant |
| asset_admin | asset_admin | 运维组 | all_tenant |
| ops_user | asset_operator | 运维组 | specified_location + specified_asset_type |
| license_admin | license_admin | 软件资产组 | specified_asset_type=software_license |
| procurement_user | procurement_user | 信息技术部 | own_org |
| maintenance_user | maintenance_user | 运维组 | specified_location |
| dept_manager | department_manager + approver | 信息技术部 | own_org_tree |
| asset_user | asset_user | 业务一部 | self |
| auditor | auditor | 信息技术部 | all_tenant |
| approver | approver | 信息技术部 | 按审批任务 |

默认字段权限：

- `tenant_admin`、`asset_admin`：资产字段可见，非系统字段可编辑，敏感字段按授权查看。
- `asset_user`：仅本人相关资产字段可见，采购金额、license key、合同附件隐藏。
- `auditor`：只读，敏感字段脱敏，不能编辑。
- `license_admin`：软件许可证字段可见，license key 默认脱敏，授权后可查看。

默认状态权限：

- `asset_admin` 可执行机房资产全生命周期动作。
- `asset_operator` 可执行 `confirm_inbound`、`deploy`、`start_operation`、`report_fault`、`finish_repair`。
- `license_admin` 可执行软件许可证登记、分配、回收、续费。
- `asset_user` 不能执行退役、处置、许可证分配。

## 6. 默认资产类型

有形资产：

- 服务器。
- 网络设备。
- 安全设备。

无形资产：

- 软件许可证。
- 证书。
- 域名。
- 云资源。

## 8. 默认生命周期模板

机房资产：

```text
planned -> purchasing -> inbound -> deployed -> running -> maintenance -> retired -> disposed
```

软件许可证：

```text
planned -> purchasing -> available -> assigned -> reclaimed -> renewal_pending -> expired -> retired
```

必须包含异常分支：

```text
maintenance -> retired
expired -> assigned
```

## 9. 默认审批模板

采购审批：

```text
申请人提交 -> 部门负责人审批 -> 资产管理员确认
```

退役审批：

```text
资产管理员提交 -> 部门负责人审批 -> 审计员留痕
```

软件授权分配审批：

```text
软件管理员提交 -> 部门负责人审批
```

审批模板必须写入 `approval_templates`、`approval_nodes`，并与 `lifecycle_transitions.approval_template_id` 建立关联。

## 10. 默认导入与测试文件

应提供：

- 服务器导入成功样例。
- 服务器导入错误样例：缺必填、枚举错误、重复 SN、未知字段。
- 软件许可证导入成功样例。
- 跨租户访问测试样例。

自动化测试不得修改演示文件本身，运行时复制到临时目录。
