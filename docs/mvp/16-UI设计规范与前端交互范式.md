# 16-UI设计规范与前端交互范式

## 1. 设计定位

本系统是企业级 IT 资产管理平台，UI 必须采用 B 端 SaaS 管理后台范式。

设计目标：

- 专业、克制、可信。
- 信息密度适中偏高，适合长期高频使用。
- 操作路径清晰，减少学习成本。
- 支持复杂表格、动态表单、审批、生命周期和报表场景。
- 视觉统一，组件复用，避免页面各自为政。

不采用：

- 营销型首页。
- 大面积渐变和装饰图形。
- 低信息密度大留白。
- 过度卡片化。
- 花哨动效。
- 与业务无关的插画。

## 2. 参考范式

设计参考以下主流设计系统的原则，但最终落地以 Vue3 + Element Plus 为基础：

- Ant Design：企业级产品设计语言，强调一致性、确定性和高效协作。
- Material Design 3：组件、样式、状态和可访问性规范完整，适合作为基础交互参考。
- IBM Carbon Design System：适合数据密集型产品，尤其是表格、筛选、批量操作和企业级视觉克制。
- Atlassian Design System：适合团队协作、任务、工作流、状态和产品一致性。

本项目不照搬任何单一设计系统，而是采用适合 ITAM 的企业后台组合范式。

## 3. 整体布局

默认布局：

```text
左侧导航
顶部工具栏
主内容区
右侧抽屉/详情区
```

左侧导航：

- 固定宽度。
- 支持分组。
- 支持图标加菜单名。
- 当前菜单高亮。
- 租户管理员和普通用户看到的菜单不同。

顶部工具栏：

- 当前租户。
- 全局搜索入口。
- 通知入口。
- 当前用户菜单。
- 帮助入口。

主内容区：

- 页面标题。
- 页面说明只在必要时出现，避免冗长描述。
- 操作栏。
- 筛选区。
- 表格或主工作区。
- 分页。

详情区：

- 优先使用右侧抽屉。
- 复杂详情可使用详情页。
- 不使用多层弹窗嵌套。

## 4. 视觉风格

整体风格：

- 背景：浅灰工作台背景。
- 内容区：白色或接近白色。
- 主色：稳定蓝色，用于主操作和选中状态。
- 成功：绿色。
- 警告：橙色。
- 危险：红色。
- 禁用：灰色。
- 信息：蓝灰或青色。

颜色使用规则：

- 主色只用于关键操作、链接、选中状态。
- 状态颜色只表达状态，不做装饰。
- 不使用大面积紫色、渐变、霓虹色。
- 表格和表单以中性色为主。

圆角：

- 卡片、抽屉、表单容器使用 6px 到 8px。
- 按钮和输入框遵循 Element Plus 默认或统一配置。

阴影：

- 页面主体不滥用阴影。
- 抽屉、下拉、浮层可使用轻阴影。

## 5. 字体与间距

字体：

- 使用系统默认无衬线字体。
- 中文环境优先保证清晰可读。
- 不使用装饰字体。

字号建议：

```text
页面标题：20px
区块标题：16px
正文和表格：14px
辅助说明：12px
数字指标：20px 到 28px
```

间距建议：

```text
页面外边距：16px 或 24px
区块间距：16px
表单项间距：16px
表格工具栏与表格间距：12px
```

要求：

- 不使用随视口变化的字体大小。
- 文本不能溢出按钮、标签、表格单元格。
- 长字段必须省略并提供 tooltip。

## 6. 组件使用规范

按钮：

- 主按钮只保留一个。
- 危险操作使用红色文字或危险按钮，并二次确认。
- 表格行内操作超过三个时使用“更多”下拉。
- 图标按钮必须有 tooltip。

表格：

- 用于资产、审批、用户、日志、导入结果等列表。
- 支持排序、筛选、分页、列设置。
- 支持行选择和批量操作。
- 重要状态使用状态徽标。
- 长文本省略并 tooltip 展示。

筛选区：

- 默认一行展示常用筛选。
- 高级筛选折叠。
- 支持重置。
- 查询按钮和重置按钮位置固定。

表单：

- 动态表单按分组展示。
- 必填字段标识清晰。
- 错误提示靠近字段。
- 复杂对象使用选择器，不让用户输入自由文本。

抽屉：

- 用于资产详情、编辑表单、审批详情。
- 宽度根据内容设置，常规 640px 到 960px。
- 底部固定操作栏。

弹窗：

- 只用于确认、短表单、危险操作。
- 不在弹窗中嵌套复杂表格。

时间线：

- 用于生命周期事件和审批记录。
- 展示动作、状态变化、操作人、时间、说明。

标签和徽标：

- 生命周期状态使用标签。
- 审批状态使用标签。
- 到期风险使用徽标。

空状态：

- 提供简短说明和明确操作入口。
- 不使用大插画。

加载状态：

- 页面级使用骨架或 loading。
- 表格使用表格 loading。
- 按钮提交时显示 loading，防止重复提交。

## 7. 核心页面范式

### 7.1 资产列表页

结构：

```text
页面标题
操作栏：新增、导入、导出、列设置
筛选区：资产类型、状态、组织、责任人、关键字
资产表格
分页
```

表格固定列：

- 资产编号。
- 资产名称。
- 资产类型。
- 生命周期状态。
- 责任人。
- 所属组织。
- 更新时间。
- 操作。

动态列由列表配置决定，无权限字段不显示。

### 7.2 资产详情页

结构：

```text
资产标题区
状态和关键操作
基础信息
动态字段
生命周期时间线
资产关系
附件
审计记录
```

动作按钮：

- 只显示当前状态可执行动作。
- 需要审批的动作显示审批标识。
- 危险动作二次确认。

### 7.3 元数据配置页

结构：

```text
左侧资产类型树
右侧配置区
  字段配置
  表单配置
  列表配置
  查询配置
```

设计要求：

- 字段编码创建后不可编辑。
- 字段类型使用选择器。
- JSON 配置区域必须有格式校验。
- 发布配置前展示影响提示。

### 7.4 审批待办页

结构：

```text
筛选区
待办表格
审批详情抽屉
审批意见区
通过/驳回按钮
```

驳回必须填写意见。

### 7.5 报表看板页

结构：

```text
关键指标卡
资产类型分布
生命周期分布
软件授权使用率
到期提醒列表
待办任务列表
```

要求：

- 不做炫酷大屏。
- 以运营决策为主。
- 图表不超过两种主色。
- 数字卡片必须可点击跳转明细。

## 8. 动态表单交互

动态表单由后端 runtime metadata 驱动。

前端渲染规则：

```text
字段定义
+ 表单 schema
+ 字段权限
+ 状态权限
+ 当前用户权限
```

字段状态：

- 可编辑。
- 只读。
- 隐藏。
- 脱敏。
- 禁用。

提交规则：

- 前端做即时校验。
- 后端做权威校验。
- 未知字段不能提交。
- 只读字段不能提交修改。

## 9. 权限可视化

权限页面必须清晰表达：

- 功能权限。
- 数据范围。
- 字段权限。
- 状态动作权限。

不要把所有规则塞进一个巨大 JSON 编辑器。

建议：

- 功能权限使用树。
- 数据范围使用表格和选择器。
- 字段权限使用资产类型加字段表格。
- 状态权限使用状态矩阵。

## 10. 响应式要求

第一版优先桌面端。

最低支持：

```text
1366x768
1440x900
1920x1080
```

笔记本宽度下：

- 表格横向滚动。
- 筛选区可折叠。
- 抽屉宽度自适应。

第一版不做移动端深度适配。

## 11. 可访问性和可用性

要求：

- 所有表单字段有 label。
- 错误提示清晰。
- 图标按钮有 tooltip。
- 危险操作二次确认。
- 键盘 Tab 顺序合理。
- 颜色不作为唯一状态表达，必须配文字。
- 加载、空、错误、无权限状态必须有明确反馈。

## 12. 前端开发约束

必须遵守：

- 使用 Element Plus 组件优先。
- 使用统一主题变量。
- 不在页面中硬编码权限结果。
- 不为每个资产类型写一套页面。
- 动态表单、动态列表、动态查询必须复用组件。
- 页面内不得出现大量解释性文字。
- 所有页面必须有 loading、empty、error 状态。
- 表格列和按钮文本不能挤压变形。

## 13. 验收标准

UI 验收必须满足：

1. 整体为企业级后台风格。
2. 页面结构一致。
3. 主流程在 1366x768 下可用。
4. 资产列表、资产详情、动态表单、审批待办、报表看板都有完整状态。
5. 字段权限在 UI 上体现为隐藏、只读、脱敏。
6. 生命周期状态和动作清晰可见。
7. 表格支持筛选、分页、排序、列设置。
8. 无权限、空数据、加载失败都有清晰提示。
9. 危险操作有确认。
10. 不出现营销式首页、过度装饰、低信息密度大留白。

## 15. 前端工程规范补充

推荐目录：

```text
src/
  api/
  components/
  composables/
  layouts/
  modules/
  router/
  stores/
  styles/
  utils/
```

工程约束：

- API 访问必须通过统一 client 封装，统一处理响应信封、错误码、traceId、401 刷新、403 无权限。
- 权限判断封装为 composable 或指令，不在页面内散落字符串判断。
- 动态表单、动态表格、动态筛选、字段脱敏展示必须做成复用组件。
- Pinia store 只保存会话、菜单、权限、租户上下文、必要的页面缓存，不保存后端权威状态。
- 表格分页、排序、筛选参数必须与 `07-API接口规格说明.md` 一致。
- 表单提交必须处理字段权限：不可编辑字段不能提交覆盖。
- 所有接口错误必须展示可理解的消息，并保留 traceId 便于排查。

组件级尺寸建议：

- 表格默认行高 40px，紧凑模式 32px。
- 主筛选区默认一行 4 个控件，超过折叠。
- 详情抽屉宽度桌面端 720px 到 960px，复杂审批详情可使用 60vw，上限 1080px。
- 主按钮每个操作区最多 1 个，其余动作进入更多菜单。
- 状态标签必须同时有颜色和文字，不能只靠颜色表达。

## 16. 关键复杂页低保真结构

资产列表：

```text
顶部标题/主操作
筛选区（可折叠）
表格工具条（列设置/导入/导出）
资产表格（分页/排序/行操作）
右侧详情抽屉
```

资产详情：

```text
标题区（资产编号/名称/状态/主动作）
基础信息分组
动态字段分组
关系/附件/审计 Tab
右侧生命周期时间线
```

元数据配置：

```text
左侧资产类型树
右侧字段表格
字段编辑抽屉
表单/列表/查询配置 Tab
发布与预览入口
```

审批详情：

```text
审批标题和当前状态
业务对象摘要
审批节点时间线
审批意见区
通过/驳回/撤销操作区
```

报表看板：

```text
顶部筛选（组织/资产类型/日期）
关键指标卡片
生命周期分布图
许可证使用率图
到期提醒和待办任务明细表
```

这些结构是开发低保真线框的最低要求，最终视觉必须保持企业级后台高信息密度和一致布局。
