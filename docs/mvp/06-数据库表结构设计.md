# 06-数据库表结构设计

## 1. 数据库选择

数据库使用 PostgreSQL。

关键原因：

- 支持 JSONB 动态字段。
- 支持 GIN、BTree、部分唯一索引和表达式索引。
- 支持复杂查询。
- 适合企业管理系统。

## 2. 公共字段与约束原则

租户业务表必须包含：

```text
id UUID PRIMARY KEY
tenant_id UUID NOT NULL
created_at TIMESTAMPTZ NOT NULL DEFAULT now()
updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
deleted BOOLEAN NOT NULL DEFAULT FALSE
```

涉及操作人的表增加：

```text
created_by UUID
updated_by UUID
```

软删除表不得使用普通唯一约束，应使用部分唯一索引：

```sql
CREATE UNIQUE INDEX ux_table_tenant_code_active
ON table_name(tenant_id, code)
WHERE deleted = false;
```

## 3. 平台与身份表

核心表：

- `tenants`
- `tenant_users`
- `organizations`
- `roles`
- `user_roles`
- `permissions`
- `role_permissions`
- `audit_logs`

关键约束：

- `tenants.tenant_code` 全局唯一。
- `tenant_users(tenant_id, username)` 在 `deleted=false` 下唯一。
- `organizations(tenant_id, org_code)` 在 `deleted=false` 下唯一。
- `roles(tenant_id, role_code)` 在 `deleted=false` 下唯一。

`tenant_users` 必备字段：

```text
id
tenant_id
username
display_name
email
mobile
password_hash
status active/disabled/locked
must_change_password boolean
last_login_at
created_by
updated_by
created_at
updated_at
deleted
```

平台用户建议单独使用 `platform_users`，避免与租户用户混用：

```text
id
username
display_name
password_hash
role_code platform_admin/platform_auditor
status
must_change_password
last_login_at
created_at
updated_at
deleted
```

## 5. 资产表

### assets

字段：

```text
id UUID PRIMARY KEY
tenant_id UUID NOT NULL
asset_no VARCHAR(64) NOT NULL
asset_name VARCHAR(200) NOT NULL
asset_kind VARCHAR(32) NOT NULL
asset_type_id UUID NOT NULL
lifecycle_status VARCHAR(64) NOT NULL
owner_user_id UUID
owner_org_id UUID
location_id UUID
cost_center_id UUID
responsible_user_id UUID
serial_no VARCHAR(128)
brand VARCHAR(128)
model VARCHAR(128)
vendor VARCHAR(128)
warranty_end_date DATE
license_end_date DATE
source_type VARCHAR(32) NOT NULL DEFAULT 'manual'
sync_source VARCHAR(64)
metadata_version INT
status VARCHAR(32) NOT NULL DEFAULT 'active'
attributes JSONB NOT NULL DEFAULT '{}'::jsonb
created_by UUID
updated_by UUID
created_at TIMESTAMPTZ NOT NULL DEFAULT now()
updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
deleted BOOLEAN NOT NULL DEFAULT FALSE
```

索引：

```sql
CREATE UNIQUE INDEX ux_assets_tenant_asset_no_active
ON assets(tenant_id, asset_no)
WHERE deleted = false;

CREATE UNIQUE INDEX ux_assets_tenant_serial_no_active
ON assets(tenant_id, serial_no)
WHERE deleted = false AND serial_no IS NOT NULL;

CREATE INDEX idx_assets_tenant_type ON assets(tenant_id, asset_type_id) WHERE deleted = false;
CREATE INDEX idx_assets_tenant_status ON assets(tenant_id, lifecycle_status) WHERE deleted = false;
CREATE INDEX idx_assets_tenant_location ON assets(tenant_id, location_id) WHERE deleted = false;
CREATE INDEX idx_assets_warranty_end ON assets(tenant_id, warranty_end_date) WHERE deleted = false;
CREATE INDEX idx_assets_license_end ON assets(tenant_id, license_end_date) WHERE deleted = false;
CREATE INDEX idx_assets_attributes_gin ON assets USING gin(attributes);
```
