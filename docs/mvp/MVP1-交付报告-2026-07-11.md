沙箱环境无 Docker / PostgreSQL，**无法执行启动校验**。请在具备 Docker 的本地环境验证：

```bash
# 1) 启动 PostgreSQL（若用 docker-compose，参考项目根 docker-compose.yml）
docker run -d --name itam-pg -p 5432:5432 \
  -e POSTGRES_DB=itam -e POSTGRES_USER=itam -e POSTGRES_PASSWORD=itam postgres:16

# 2) 后端（Flyway 自动执行 V1~V4；context-path=/api，端口 8080）
cd backend && mvn spring-boot:run
# 或：java -jar target/itam-backend-0.0.1.jar

# 3) 前端（开发态）
cd frontend && npm install && npm run dev   # 默认 5173，代理 /api → :8080

# 4) 真实 DB 集成测试（可选）
ITAM_INTEGRATION=1 mvn test   # 需 Testcontainers 或本地 PG
```

## 十一、已知问题 / MVP-2+ 阻塞项

1. **字段权限配置 UI 延后**：`field_permission_rules` 表已建但无配置界面（按规划延至 MVP-3）；当前用内存矩阵，API 响应已正确反映过滤。（设计预期，非缺陷）

4. **元数据端点收敛**：doc 07 第 15 个端点（单资产类型详情）合并进 tree 节点，未单列。
5. **前端未做 E2E 浏览器测试**：沙箱无头浏览器，仅 `npm run build` 类型 + 构建通过；交互层建议在本地手工走查。
6. **Docker 启动未校验**：见第九节，需用户在本地环境执行。
