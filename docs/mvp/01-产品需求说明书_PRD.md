# 01-产品需求说明书 PRD

## 1. 产品定位

本系统定位为多租户、元数据驱动、生命周期优先的 IT 资产管理平台。

第一版客户画像是单企业或集团内部的多组织、多地点、多资产类型管理场景。系统从第一天支持租户隔离和平台侧管理能力，但第一版不做公开 SaaS 化运营、计费和租户自助开户。

系统覆盖有形资产和无形资产：

- 有形资产：服务器、网络设备、安全设备、办公终端、备件耗材等。
- 无形资产：软件许可证、证书、域名、云资源、SaaS 订阅等。

第一版以机房资产和软件许可证为完整生命周期样板，证明统一资产模型、动态字段、权限、审批、审计和报表能力可用。

## 2. 建设目标

建设目标：

1. 替代临时性资产台账和非规范化旧系统。
2. 支持企业内部多部门、多地点、几千到几万资产规模。
3. 从第一天支持多租户，预留集团级扩展。
4. 使用元数据支持字段、表单、列表、查询配置。
5. 用生命周期状态机驱动资产管理，而不是只做静态台账。
6. 为后续云同步、自动发现、CMDB、工单系统集成预留接口。

第一版北极星指标：

- 资产台账覆盖率：演示租户核心资产样例 100% 进入平台。
- 生命周期可追踪率：机房资产和软件许可证关键流转 100% 写入事件。
- 权限命中率：跨租户访问、无权限字段、无权限动作均能被自动化测试拦截。
- 操作可审计率：关键写操作 100% 写入审计日志。

## 3. 用户角色

平台侧：

- 平台管理员：创建租户、停用租户、查看平台状态。
- 平台审计员：查看平台级审计，不参与租户业务。

平台角色没有租户上下文，不能直接读取或操作租户业务数据。确需平台托管访问时，应作为后续受控功能单独设计审批、授权和审计。

租户侧：

- 租户管理员：管理本租户组织、用户、角色、资产类型、字段、流程模板。
- 资产管理员：管理资产类型、资产台账、生命周期操作。
- 机房运维人员：管理指定地点和指定类型的机房资产。
- 软件管理员：管理软件许可证、授权分配、回收、续费。
- 采购人员：提交和维护采购申请。
- 维修人员：处理故障和维修记录。
- 部门负责人：查看本部门及下级资产，处理审批。
- 普通使用人：查看自己名下资产和待处理事项。
- 审计员：查看审计日志和只读报表。
- 审批人：处理分配给自己的审批任务。

`approver` 是职能角色，不能单独代表任意审批权。审批动作必须同时满足审批任务归属、审批节点规则、数据范围和任务状态。

## 4. MVP 范围

MVP 分为六个阶段：

```text
MVP-0 平台底座
MVP-1 元数据与资产核心
MVP-2 生命周期状态机与动作（不含审批）
MVP-3 审批、权限增强、通知
MVP-4 导入导出、报表、轻量资产扩展
MVP-5 生产化加固与集成
```

完整生命周期样板：

- 机房资产：服务器、网络设备、安全设备。
- 软件许可证：商业软件授权、数据库授权、中间件授权、操作系统授权。

轻量管理样板：

- 证书。
- 域名。
- 云资源。

## 5. MVP 不做范围

第一版不做：

- 旧系统历史数据迁移。
- BPMN 流程设计器。
- 完整低代码平台。
- 云厂商自动同步。
- 自动资产扫描。
- 复杂 CMDB 拓扑图。
- 每租户独立数据库。
- SaaS 计费。

## 6. 核心业务流程

采购申请是独立业务对象，不是资产创建的唯一入口。MVP-1 允许资产管理员直接创建资产并进入初始状态；采购流程在 MVP-2/MVP-3 作为生命周期动作和审批的业务来源。

机房资产流程：

```text
采购申请 -> 到货入库 -> 上架部署 -> 运行 -> 故障维修 -> 退役审批 -> 处置
```

软件许可证流程：

```text
采购申请 -> 授权登记 -> 分配使用 -> 回收 -> 续费 -> 停用
```

证书/域名流程：

```text
登记 -> 责任人绑定 -> 业务绑定 -> 到期提醒 -> 续费记录 -> 停用
```

云资源流程：

```text
登记 -> 归属绑定 -> 成本中心绑定 -> 状态维护 -> 退役
```

## 7. 验收目标

第一版验收时必须证明：

1. 平台管理员可以创建租户。
2. 租户管理员可以配置组织、用户、角色、资产类型、字段、表单、列表。
3. 机房资产可跑完整生命周期。
4. 软件许可证可跑完整生命周期。
5. 不同角色看到的数据和字段不同。
6. 审批通过后触发生命周期流转。
7. 审批驳回、撤销后资产状态不变。
8. 导入导出遵守字段权限。
9. 到期、待审批、续费、退役等事项可提醒。
10. 所有关键操作有审计日志。
11. 跨租户访问、状态越权、字段越权、并发唯一冲突都有自动化测试覆盖。
