# 12-测试用例与验收标准

## 1. 测试分层

测试分为：

- 单元测试。
- 集成测试。
- API 契约测试。
- 端到端测试。
- 权限和安全测试。
- 并发和一致性测试。
- 性能基准测试。
- 验收测试。

每个用例必须包含前置条件、步骤、期望结果和通过标准。不能只写场景名称。

## 4. MVP-2 验收：生命周期状态机与动作（不含审批）

| 编号 | 前置条件 | 步骤 | 期望结果 | 通过标准 |

## 10. 性能门禁

初始性能目标：

- 资产列表在 50,000 条资产、常用筛选条件下 P95 小于 500ms。
- 资产详情 P95 小于 300ms。
- 审批待办列表 P95 小于 500ms。
- 导入 10,000 行文件必须进入异步任务，不得阻塞 HTTP 请求直到全部处理完成。
- 报表分页接口 P95 小于 800ms。
- PostgreSQL 常用查询必须命中 `tenant_id` 和业务索引，不允许全表扫描成为默认路径。

## 11. 质量门禁

每个阶段必须满足：

- 后端单元和集成测试通过。
- 前端构建通过。
- 数据库迁移可重复执行。
- OpenAPI 文档可访问。
- API 契约测试通过。
- 权限测试通过。
- 租户隔离测试通过。
- 当前阶段验收用例全部通过。

第一版最低覆盖率目标：

- 后端核心模块行覆盖率不低于 60%。
- 权限、生命周期、审批、许可证分配的关键路径必须有集成测试。
- P0 安全用例不得失败。
