# 02-MVP功能清单与用户故事

## 1. MVP-0 平台底座

功能清单：

- Git 工程初始化。
- Docker Compose：PostgreSQL、Redis。
- Spring Boot 后端工程。
- Vue3 TypeScript 前端工程。
- Flyway 数据库迁移。
- 租户、用户、组织、角色、权限、审计基础表。
- JWT 登录、刷新、退出和租户上下文。
- 平台用户与租户用户上下文隔离。
- OpenAPI 文档。
- 健康检查接口。

用户故事：

- 作为平台管理员，我可以创建租户，以便不同组织独立使用系统。
- 作为租户管理员，我可以登录系统，以便进入本租户管理后台。
- 作为开发人员，我可以通过 Docker Compose 启动依赖，以便本地开发和测试。

## 2. MVP-1 元数据与资产核心

功能清单：

- 资产类型树。
- 字段定义。
- 表单配置。
- 列表配置。
- 查询配置。
- 资产新增、编辑、详情、列表。
- JSONB 动态字段。
- 热点字段物理列和唯一索引策略。
- 字段权限基础能力：可见、可编辑、脱敏、可导出。
- 资产关系。
- 服务器、网络设备、安全设备、软件许可证预置元数据。

用户故事：

- 作为租户管理员，我可以创建资产类型，以便管理不同资产。
- 作为租户管理员，我可以为资产类型配置字段，以便不用修改代码就扩展字段。
- 作为资产管理员，我可以创建服务器资产，以便记录机房设备。
- 作为软件管理员，我可以创建软件许可证资产，以便记录授权信息。
- 作为审计员，我访问资产详情时看不到无权限字段，以便满足最小可见原则。

## 3. MVP-2 生命周期状态机与动作（不含审批）

功能清单：

- 生命周期模板。
- 生命周期状态。
- 生命周期流转。
- 生命周期动作 API。
- 守卫规则。
- 生命周期事件日志。
- 前端动作面板和时间线。

边界说明：

- MVP-2 不实现人工审批、审批任务和审批通知。
- MVP-2 的目标是验证状态机、动作、守卫规则、事件日志和前端时间线。
- 标记 `require_approval=true` 的动作在 MVP-2 可配置为免审批执行，以便阶段独立验收。
- 从 MVP-3 起，同一动作会创建审批实例，审批通过后再执行状态流转。

用户故事：

- 作为资产管理员，我可以通过动作让服务器从采购进入入库，以便保证状态变化可追踪。
- 作为机房运维人员，我可以将设备上架并进入运行状态。
- 作为维修人员，我可以将运行设备标记为维修中，并在维修完成后恢复运行。
- 作为软件管理员，我可以将许可证从可分配变成已分配。

## 4. MVP-3 审批、权限增强、通知

功能清单：

- 审批模板。
- 审批节点。
- 审批实例。
- 审批任务。
- 生命周期动作与审批模板绑定。
- 数据范围权限。
- 字段权限增强。
- 状态权限。
- 站内通知。

用户故事：

- 作为审批人，我可以审批采购申请，以便控制资产进入流程。
- 作为部门负责人，我只能审批自己范围内的任务。
- 作为普通使用人，我只能查看自己名下资产。
- 作为软件管理员，我可以分配许可证，但不能退役机房设备。

## 5. MVP-4 导入导出、报表、轻量资产

功能清单：

- Excel/CSV 导入模板。
- Excel/CSV 导入校验。
- 异步导入任务和行级错误。
- Excel 导出。
- 资产概览。
- 生命周期统计。
- 软件授权使用率。
- 到期提醒。
- 待办任务报表。
- 证书、域名、云资源轻量管理。

用户故事：

- 作为资产管理员，我可以批量导入资产，以便快速建立台账。
- 作为审计员，我可以查看资产概览，以便掌握资产总体情况。
- 作为软件管理员，我可以查看许可证使用率，以便发现授权风险。
- 作为责任人，我可以收到证书或许可证到期提醒。

## 6. MVP-5 生产化加固与集成

功能清单：

- API Token。
- Webhook。
- Webhook HMAC 签名和失败重试。
- 附件权限。
- 敏感字段脱敏和加密存储。
- 缓存失效。
- 备份恢复脚本。
- 部署文档。
- 运维检查清单。

用户故事：

- 作为集成系统，我可以使用 API Token 查询资产。
- 作为管理员，我可以配置 Webhook，以便资产事件通知外部系统。
- 作为安全管理员，我可以确保许可证密钥等敏感字段不泄露。
- 作为运维人员，我可以备份和恢复数据库。
